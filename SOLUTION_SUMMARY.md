## 问题 1️⃣ — "为什么刷新网页以后同个人还可以投票？"

**原因：** 原系统只在浏览器本地存储投票，刷新页面就丢失了

**解决方案：**
- ✅ 添加 **Node.js 后端服务器** 存储投票数据
- ✅ 投票数据保存到 `votes.json` 文件（永久存储）
- ✅ 投票页面加载时**自动检查后端**是否已投票
- ✅ 已投票用户刷新后**继续无法投票**

**新工作流程：**
```
用户选择名字 → 前端检查后端 → 如已投票则禁用UI
用户投票 → 提交到后端 → 后端存储 + 前端禁用
用户刷新 → 再次检查后端 → 看到"已投票"状态 → 无法再投
```

## ❓ 常见疑问

**Q: 投票数据存在哪？**
A: 存在 `votes.json` 文件，你可以随时打开查看或备份

**Q: 想清空所有投票数据？**
A: 删除 `votes.json` 文件即可，系统会自动重新生成

**Q: 支持多少人同时投票？**
A: 理论上无限制，系统采用 JSON 存储，可支持几百甚至几千人

**Q: 投票可以修改吗？**
A: 原系统不支持修改，但你可以直接编辑 `votes.json` 文件后端会重新计算统计

**Q: 后端崩溃了怎么办？**
A: 数据保存在 `votes.json` 文件，重启 `node server.js` 即可恢复
